# Pijamas Guidelines

This repository contains the official development, contribution, and collaboration guidelines for the Pijamas organization.

The purpose of this repository is to provide a centralized location for team standards, workflows, and best practices used across all Pijamas projects.

## Why This Repository Exists

As the organization grows and manages multiple repositories, maintaining a consistent workflow becomes increasingly important.

This repository helps ensure:

* Consistent Git workflows
* Clear contribution processes
* Standardized Pull Request and Issue handling
* Shared coding and documentation practices
* Easier onboarding for new contributors

## Documents

### Contribution Guidelines

* [CONTRIBUTING.md](./CONTRIBUTING.md)

Defines the contribution process, expectations, and collaboration rules for all contributors.

## Repository Usage

All repositories within the Pijamas organization should follow the standards defined in this repository unless project-specific requirements state otherwise.

Project repositories are encouraged to reference this repository from their own README or CONTRIBUTING documentation.

## Future Documents

As the organization evolves, additional guidelines may be added, including:

* Architecture Guidelines
* Security Practices
* Release Management
* Documentation Standards
* API Design Guidelines
